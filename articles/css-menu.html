Mobile Hamburger Menu with only CSS (and HTML)

The second part of this portfolio was to style it with some CSS. In addition to the overall asethetic, flexboxes, and custom fonts, one of the things I was tasked with exploring was responsive design. I used a fluid type scale calculator provided by Utopia.fyi to have the text feel more fluid and responsive.

Another thing I utilized were breakpoints to determine the layout of the header for desktop, tablet, and mobile displays. For my mobile layout, I first tried a grid display with 2 columns, but it felt rather clunky and space inefficient. This is when I decided to look into making a typical mobile hamburger menu that you'd normally see on everyday sites. However, I wasn't supposed to use any JavaScript for this part which meant I had to make this menu with pure CSS (and HTML of course). Of course, there are already quite a few examples out there, so I knew this was possible. The one that I particularly liked and used as inspiration is by Janessa Garrow, of which you can find here.

The HTML

<header>
        <h2>...</h2>
        <menu-button>
                <input type="checkbox" id="menuButton"" name="menuButton">
                <label for="menuButton">
                        <svg>
                                ⋮
                        </svg>
                </label>
        </menu-button>
        <nav>
                ⋮
        </nav>
</header>

The HTML itself is very basic with a heading and a nav tag. I use a custom "menu-button" tag, for semantic purposes, to serve as the wrapper for the hamburger menu. I use a checkbox input to serves as a way to keep track the state of whether or not the user has opened/closed the menu. The label is wrapped around the SVG, which is the hamburger icon, so that pressing on the SVG will also trigger the checkbox. However, we don't want to actually show the checkbox itself and we want to make sure that we only display the menu for mobile layouts, to which brings us to the CSS portion.

The CSS

nav {
        display: flex;
        flex-direction: column;
        overflow: hidden;
        max-height: 0;
}

nav > a {
        pointer-events: none;
}

input#menuButton {
        display: none;
}
menu-button:has(#menuButton:checked) ~ nav{
    max-height: 100%;
}

menu-button:has(#menuButton:checked) ~ nav > a{
    pointer-events: auto;
}

The key part of this CSS that makes the menu work is the max-height property. We set the nav max-height to 0 to hide the nav menu by default (aka the checkbox is not checked) and we allow the nav menu to expand to fit its elements when it is checked.
